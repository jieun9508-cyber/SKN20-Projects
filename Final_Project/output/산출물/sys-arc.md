1. 머메이드 변환 (1단계): 시각적 정보를 구조화된 데이터로 바꾸는 첫 단추입니다.

2. 상세 설명 (2단계): 도형에 담기지 않은 사용자의 '의도'를 파악하는 단계입니다.

3. 질문 및 꼬리질문 (3단계): * 구글 문서를 기반으로 하되, 전문 용어를 상황(Scenario)으로 번역하여 질문합니다.
질문을 통해 평가에 필요한 부족한 정보를 수집합니다.

4. 6대 지표 개별 평가 (4단계): 독립된 에이전트가 병렬로 평가하여 전문성과 객관성을 확보합니다.

5. 프롬프트 변환 (5단계): * 전문을 그대로 읽히지 않고 '핵심 원칙 - 질문 전략' 형태로 정제하여 주입합니다.

6. 지표 정보 공통 사용 (6단계): 질문자가 물어본 내용을 평가자가 채점하는 논리적 일관성을 유지합니다.

7. 맥락 유지 (7단계): * 고정 맥락: 아키텍처 도형 + 첫 설명
- 유동 맥락: Q&A 대화 요약 + 새롭게 파악된 사실(Fact)
- 이 정보들을 세션 동안 유지하여 "나의 설계를 이해하고 대화한다"는 느낌을 줍니다.
